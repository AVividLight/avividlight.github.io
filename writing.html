Three Years Later

Written by Michael Bethke, August 29, 2020

Three years ago, I did not expect to create 80% of a personal website and then essentially abandon it to the winds of time. I also did not expect that I would find a use for it again while locked in my home, taking a computer security course online amidst a global pandemic. But, here we are.

For my class, I've been instructed to use this blog as a place to note, discuss, and comment on websites that I find useful or interesting (within the general field of computer-security). To that end, I will read, discuss, and report on at least one site that contains useful resources/information related to the general topic of computer security every week.

So without further ado, my report for week one:

You Can Use Any Security Suite That You Want, As Long As It's Windows Defender

Source: Parmar, Mayank. “Microsoft Really Doesn't Want You to Disable Windows 10 Defender.” Windows Latest, 19 Aug. 2020, www.windowslatest.com/2020/08/19/microsoft-retires-disableantispyware-for-windows-defender/.

As of the August 2020 Windows 10 update, Microsoft has removed a registry edit which allowed users to disable the auto-start feature for Windows Defender. Making that change in your Windows registry was previously the only way to disable the built-in virus protection, however, citing the ability of malware to edit the same registry value, Microsoft has chosen to eliminate the possibility altogether.

They say that this will improve security for users, however I am concerned that there are many cases where Windows Defender may need to be disabled for legitimate reasons. An example of one such situation is using a third-party anti-virus program. Windows Defender is known for behaving poorly when software that's also trying to detect and remove malware is running in parallel.

In an attempt to alleviate concern about this issue, Microsoft has stated that Windows Defender will detect when another anti-virus program is running and will disable itself. But with that, it seems like we're back in square one; instead of editing the registry to disable Windows Defender, new malware will just trigger whatever the criteria is to disable Windows Defender and go about its nefarious business.

Perhaps Microsoft is truly confident in their new method of detecting other security suites and turning their own off. I'm not convinced, though, and I wouldn't be surprised if they may have ulterior anti-competitive motives. The new method may not work 100% of the time, and could end up unfairly hampering third-party developers. And, if that is the case, Microsoft has actually made security worse.

It's important to note that this is purely speculation. Indeed, Microsoft may simply be attempting to improve the security of millions of users, but it seems unrealistically optimistic to trust that detecting and automatically disabling anti-virus software isn't a risk.

Computer security is very much an arms race between malware authors and security developers, and the ever-changing nature of computer science means that there is no permanent solution to computer security. However, reducing user choice and hindering third-party developers does not seem like a positive change.

You Are Here

Written by Michael Bethke, August 8, 2017

Welcome to my new website! Hooray! Since I'm still finishing things up, please excuse the current state of things, and accept a brief textual-tour as a token of my truthful apology.

First things first, what is this website for? Such a straightforward question requires a straightforward answer; this website will serve as my platform for blatant self-promotion via photography, writing, and code. I am not a professional photographer, but I like taking pictures. I am not a professional author (obviously), but I like writing. I am not a professional software engineer, but I like programming. You will see the fruits of my hobbies here. That is what my website is for.
